# CV Pipeline ‚Äî Frontend ‚Üî Backend Integration Plan

**Date:** 2026-03-11
**Status:** Ready for implementation

---

## Principle

The backend is the source of truth. The frontend adapts to backend data shapes ‚Äî not the reverse.

- **Personal info** (name, email, job_title) ‚Üí comes from `citizen_profiles` DB table
- **CV analysis** (experience, skills, tools, roles) ‚Üí comes from `CVAnalysisResult`
- **Summary** ‚Üí generated by the CV analyzer agent (new field)
- **Education** ‚Üí extracted by the CV analyzer agent (new field)

---

## Backend Changes Required

### 1. Add `education` and `summary` to agent extraction

**Files to change:**
- `backend/core/cv_pipeline/schemas.py` ‚Äî add `EducationEntry` model, add `education` and `summary` fields to `PageAnalysis` and `CVAnalysisResult`
- `backend/agents/cv_analyzers/prompts.py` ‚Äî update prompt to extract education entries and generate a 1-2 sentence professional summary
- `backend/core/cv_pipeline/components/aggregator.py` ‚Äî merge education across pages (dedup by institution+degree)
- `backend/db/models/cv_upload.py` ‚Äî add `education` (JSONB) and `summary` (Text) columns to `CVVersion`
- `backend/core/cv_pipeline/db_persist.py` ‚Äî persist education and summary

**New schema:**
```python
class EducationEntry(BaseModel):
    institution: str
    degree: str = ""
    year: str = ""

class CVAnalysisResult(BaseModel):
    experience: list[ExperienceEntry]
    skills: list[str]
    soft_skills: list[str]
    tools: list[str]
    roles: list[str]
    education: list[EducationEntry]  # NEW
    summary: str = ""                # NEW
    page_count: int = 0
```

### 2. Include citizen profile in job status response

**File:** `backend/api/routers/cv.py`

The `GET /cv/jobs/{job_id}` endpoint should also return `citizen_id` so the frontend can fetch the citizen profile separately via `GET /api/citizen/profile`. No need to bundle personal info into the CV result ‚Äî it's already available through the existing profile endpoint.

**File:** `frontend/src/lib/types/cv.ts`

export interface ExperienceEntry {

**Files:** `context/types.ts`, `slices/cvSlice.ts`, `context/initialState.ts`

New actions:

### 4. Wire `UploadZone.tsx`

Replace the mock flow in `startAnalysis()`:

```
Current (mock):
  setTimeout ‚Üí MOCK_CV_DATA ‚Üí SET_CV_DATA

New (real):
  1. uploadCv(file, citizenId) ‚Üí get job_id
  2. dispatch SET_CV_JOB(job_id)
  3. streamJobProgress(job_id, (event) => {
       dispatch SET_CV_PROGRESS(event.progress_pct, event.stage)
       if (event.status === "completed") ‚Üí getJobStatus(job_id) ‚Üí dispatch SET_CV_RESULT
       if (event.status === "failed") ‚Üí show error
     })
```

### 5. Update `AnalysisProgress.tsx`

## Data Sources Summary

| Data | Source | Endpoint |
|------|--------|----------|
| Name, email, job title | `citizen_profiles` DB table | `GET /api/citizen/profile` |
| Experience, skills, tools, roles | CV analyzer agent | `GET /api/cv/jobs/{id}` ‚Üí `result` |
| Education | CV analyzer agent (new) | `GET /api/cv/jobs/{id}` ‚Üí `result.education` |
| Summary | CV analyzer agent (new) | `GET /api/cv/jobs/{id}` ‚Üí `result.summary` |
| Analysis progress | Celery worker ‚Üí Redis pub/sub | `GET /api/cv/jobs/{id}/stream` (SSE) |

## Implementation Order

1. **Backend: extend schemas + agent prompt** (add education, summary)
2. **Backend: update DB model + persistence** (new JSONB columns)
3. **Frontend: replace types** (CvData ‚Üí CVAnalysisResult)
4. **Frontend: create cvService.ts** (API + SSE client)
5. **Frontend: update state** (new actions, new shape)
6. **Frontend: wire UploadZone + AnalysisProgress** (real upload + live progress)
7. **Frontend: update display components** (adapt to new field names)
8. **Frontend: remove mock data**
